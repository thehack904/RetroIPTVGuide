# CONTRIBUTING.md

Thank you for your interest in contributing to RetroIPTVGuide.

This project is maintained as an open-source IPTV guide platform designed for retro-style television guide experiences. Contributions are welcome and encouraged.

AI-assisted contributions are welcome, but all submissions must be reviewed for correctness and security before submission.

---

## Development Workflow

1. Fork the repository.
2. Create a feature branch.
3. Implement your changes.
4. Test your changes locally.
5. Submit a pull request.

Branch naming examples:

feature/new-virtual-channel  
fix/epg-parsing-error  
docs/update-architecture

---

## Code Style

General expectations:

- Keep code readable and maintainable.
- Prefer small, focused functions.
- Avoid unnecessary dependencies.
- Document complex logic.

Python guidelines:

- Follow PEP8 where practical.
- Use clear variable names.
- Include comments for non‑obvious behavior.

---

## Pull Request Guidelines

Pull requests should:

- Describe the change clearly.
- Reference related issues.
- Avoid unrelated changes.
- Include testing notes when applicable.

---

## Reporting Issues

When submitting an issue include:

- RetroIPTVGuide version
- installation type (Docker / Linux / Windows / Raspberry Pi)
- playlist provider
- logs if available
- reproduction steps

This helps maintainers diagnose issues faster.

---

## Security Issues

If you discover a security vulnerability, please report it privately instead of opening a public issue.
